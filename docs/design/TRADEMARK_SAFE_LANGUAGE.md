# Trademark-safe language

## Hard avoid (product UI, README blurb, voice lines, store listing)

Do **not** use as product descriptors:

- Names of classic home computer platforms from the 1980s–90s demo culture (the ones with the famous bouncing checkerboard ball)
- Names of desktop video switcher products from that era
- Their logos, box art, or slogan copy

## Preferred vocabulary

| Instead of implying the old names out loud | Say |
|--------------------------------------------|-----|
| Old workstation nostalgia | **desktop-video spirit**, **live desk**, **broadcast mischief** |
| Old switcher | **TOASTED**, **the mixer**, **PGM / WORLD / PLATE**, **wet/dry** |
| Famous demo ball | **Bounce**, **checker orb**, **the pilot** |
| Old HMD experiments | **Cardboard-compatible**, **stereo eyes**, **passthrough mix** |

## Research docs

Historical design notes may reference real publications and hardware **as history** in `SOURCE_PAPERS.md` only when needed for accuracy.  
That does not license those names for the **product personality** or marketing.

## Code identifiers

Avoid public class names that are just the forbidden trademarks.  
OK: `ToastedTheme`, `BounceCursor`, `ProgramBus`, `WetDryMix`.  
Not OK: public `Amiga*`, `VideoToaster*`, etc.
